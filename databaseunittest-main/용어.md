# 📘 데이터베이스 개론 — 핵심 용어 요약집

---

# 🔷 1. 데이터베이스 기본 개념

### **데이터베이스(Database, DB)**

체계적으로 저장된 데이터의 집합.
중복을 최소화하고, 여러 사용자가 공유하도록 설계된 구조화된 데이터 저장소.

### **DBMS (Database Management System)**

데이터베이스를 생성·관리·조회·수정하는 소프트웨어.
예: MySQL, Oracle, PostgreSQL, SQL Server

### **데이터 모델(Data Model)**

현실 세계의 데이터를 구조적으로 표현하는 방식.

---

# 🔷 2. 모델링(Modeling) 용어

### **개념적 모델(Conceptual Model)**

시스템이 저장해야 할 개체(Entity)와 관계(Relationship)를 추상적으로 표현한 모델.
업무 중심, DBMS 독립적. (예: ERD)

### **논리적 모델(Logical Model)**

릴레이션·속성·키를 정확히 정의한 모델.
정규화 관점에서 설계됨.

### **물리적 모델(Physical Model)**

실제 DBMS에 저장될 테이블 구조, 데이터 타입, 인덱스 등을 구체적으로 명시.

---

# 🔷 3. 스키마(Schema)와 데이터 구조

### **스키마(Schema)**

데이터베이스의 구조를 정의한 설계 문서.
테이블 구조, 데이터 타입, 제약조건 등 **데이터의 정적 구조**.

### **인스턴스(Instance)**

스키마 구조 안에 **실제로 저장된 데이터의 상태**.
스키마는 고정, 인스턴스는 계속 변함.

---

# 🔷 4. 릴레이션(Relation) 관련 용어 (관계형 DB 핵심)

### **릴레이션(Relation)**

관계형 데이터베이스에서의 "테이블" 개념.
튜플의 집합이며, 순서가 없음.

### **튜플(Tuple)**

릴레이션의 한 행(Row).

### **속성(Attribute)**

릴레이션의 한 컬럼(Column).

### **도메인(Domain)**

속성이 가질 수 있는 값의 집합.
예: 나이(age)의 도메인 = 정수 0~120

### **카디널리티(Cardinality)**

릴레이션의 **행(row) 개수** OR
컬럼 값의 다양성(중복 없음) 의미 두 가지가 있음.

### **차수(Degree)**

릴레이션의 속성(컬럼) 개수.

---

# 🔷 5. 키(Key)

### **Primary Key (기본키)**

튜플을 유일하게 식별하는 속성.

### **Candidate Key (후보키)**

PK로 선택될 수 있는 모든 속성 집합.

### **Alternate Key (대체키)**

후보키 중 PK로 선택되지 않은 키.

### **Foreign Key (외래키)**

다른 릴레이션의 PK를 참조하는 속성.

### **Super Key (슈퍼키)**

튜플을 식별할 수 있는 속성들의 집합(PK + 다른 속성 포함 가능).

---

# 🔷 6. 관계(Relationship)

### **1:1 관계**

한 엔터티가 다른 엔터티의 한 개와만 관계.

### **1:N 관계**

가장 흔함. 한 엔터티가 여러 엔터티와 관계.

### **N:M 관계**

중간 테이블(Bridge Table)이 필요.

---

# 🔷 7. 무결성(Integrity)

### **개체 무결성(Entity Integrity)**

PK는 NULL이 될 수 없고 중복될 수 없음.

### **참조 무결성(Referential Integrity)**

FK는 반드시 부모 테이블의 PK 값을 참조해야 함.

### **도메인 무결성(Domain Integrity)**

속성 값은 정의된 도메인 범위 내여야 함.

---

# 🔷 8. 정규화(Normalization)

### **1NF (원자성)**

컬럼은 더 이상 나눌 수 없는 원자값만 가짐.

### **2NF (부분 종속 제거)**

복합키의 일부에 종속된 속성 제거.

### **3NF (이행 종속 제거)**

PK가 아닌 다른 속성에 종속된 속성 제거.

### **BCNF**

결정자는 모두 후보키여야 한다.

정규화 목적:
중복 제거 → 이상(Anomaly) 방지 → 구조적 일관성 확보

---

# 🔷 9. SQL 관련 용어

### **DDL (Data Definition Language)**

스키마 정의/변경
`CREATE`, `ALTER`, `DROP`, `TRUNCATE`

### **DML (Data Manipulation Language)**

데이터 조회/삽입/수정/삭제
`SELECT`, `INSERT`, `UPDATE`, `DELETE`

### **DCL (Data Control Language)**

권한 제어
`GRANT`, `REVOKE`

### **TCL (Transaction Control Language)**

트랜잭션 제어
`COMMIT`, `ROLLBACK`, `SAVEPOINT`

---

# 🔷 10. 트랜잭션(Transaction)

### **트랜잭션(Transaction)**

DB의 논리적 작업 단위.
원자성, 일관성, 고립성, 지속성(ACID)을 만족해야 함.

### **ACID**

* **Atomicity** (원자성)
* **Consistency** (일관성)
* **Isolation** (고립성)
* **Durability** (지속성)

---

# 🔷 11. 인덱스(Index)

### **인덱스(Index)**

검색 성능 향상을 위해 사용하는 자료구조(B-Tree 등).
WHERE, JOIN, ORDER BY에서 효과적.

### **클러스터드 인덱스(Clustered Index)**

실제 데이터가 인덱스 순서로 저장됨.

### **논클러스터드 인덱스(Non-Clustered Index)**

데이터는 원래 위치, 인덱스만 별도 구조.

---

# 🔷 12. 뷰(View)

### **View**

SELECT query를 저장한 가상의 테이블.
데이터는 저장하지 않음. 보안/단순화 용도.

---

# 🔷 13. 조인(Join)

### **INNER JOIN**

양쪽 테이블 모두 매칭되는 행만 선택.

### **LEFT JOIN**

왼쪽 테이블 전체 + 오른쪽 매칭 데이터.

### **RIGHT JOIN**

오른쪽 전체 + 왼쪽 매칭 데이터.

### **FULL OUTER JOIN**

양쪽 모든 행 포함 (MySQL은 미지원)

### **CROSS JOIN**

모든 조합(N × M)

---

# 🔷 14. 쿼리 결과(Result Set)

### **Result Set**

SELECT의 출력 결과.
행(row)과 열(column)로 구성된 가상 테이블.

---

# 🔷 15. 기타 기본 개념

### **카탈로그(Catalog)**

DBMS가 관리하는 시스템 메타데이터 저장소.

### **데이터 사전(Data Dictionary)**

테이블, 인덱스, 사용자 정보 등 “DB 구조 정보” 저장.

### **커서(Cursor)**

Result Set을 한 row씩 순회하기 위한 포인터.

### **트리거(Trigger)**

INSERT/UPDATE/DELETE 시 자동 실행되는 프로시저.

### **스토어드 프로시저(Stored Procedure)**

DB 서버에 저장된 SQL 로직 묶음.

---

# ✨ 최종 정리

입문자가 반드시 알아야 하는 핵심 용어는 다음 5개입니다:

* **스키마(Schema)**: DB 구조 설계
* **릴레이션(Relation)**: 테이블
* **튜플(Tuple)**: 행
* **속성(Attribute)**: 컬럼
* **정규화(Normalization)**: 데이터 구조 품질 관리
