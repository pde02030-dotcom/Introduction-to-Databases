### 2. DBMS의 정의와 주요 기능

| 슬라이드 | 제목 | 내용 구성 |
| :--- | :--- | :--- |
| **S 2.1** | **DBMS (Database Management System) 정의** | * **정의:** 사용자 또는 응용 프로그램이 데이터베이스를 공유하고, 정의하고, 질의하고, 저장하고, 보호할 수 있도록 해주는 **소프트웨어 시스템**. |
| **S 2.2** | **DBMS의 3대 핵심 기능** | *  |
| | | 1. **정의 (Definition):** 데이터베이스 구조(스키마) 및 제약 조건 명세. (데이터 정의어, **DDL**) |
| | | 2. **조작 (Manipulation):** 데이터 검색, 삽입, 삭제, 갱신 등 처리. (데이터 조작어, **DML**) |
| | | 3. **제어 (Control):** 데이터 무결성, 보안, 권한, 동시성 등을 유지. (데이터 제어어, **DCL**) |
| **S 2.3** | **1. 데이터 정의 기능 (DDL)** | * **역할:** 데이터의 **논리적 구조(스키마)**와 데이터 간의 관계를 명세. |
| | | * **주요 명령어:** `CREATE`, `ALTER`, `DROP`. |
| **S 2.4** | **2. 데이터 조작 기능 (DML)** | * **역할:** 사용자가 데이터를 검색하고 접근하게 해주는 인터페이스 제공. |
| | | * **주요 명령어:** `SELECT`, `INSERT`, `UPDATE`, `DELETE`. |
| **S 2.5** | **3. 데이터 제어 기능 (DCL)** | * **역할:** 시스템의 안정성과 보안을 유지. |
| | | * **주요 항목:** **보안(권한 부여), 무결성 유지(제약 조건 검사), 동시성 제어(트랜잭션)**. |

### 3. DBMS의 장점과 단점

| 슬라이드 | 제목 | 내용 구성 |
| :--- | :--- | :--- |
| **S 3.1** | **DBMS의 장점 (파일 시스템 대비)** | * **중복 최소화:** 모든 데이터를 통합 관리하여 중복을 줄임. |
| | | * **데이터 공유:** 여러 사용자 및 응용 프로그램이 데이터 공동 사용. |
| | | * **무결성 유지:** 일관성, 정확성, 유효성을 위한 제약 조건 강제. |
| | | * **보안 강화:** 접근 권한을 세밀하게 제어 가능. |
| | | * **데이터 독립성 보장:** (챕터 3에서 자세히 다룰 예정) |
| **S 3.2** | **DBMS의 단점** | * **비용 부담:** DBMS 소프트웨어 도입 비용 및 운영 인력 비용 높음. |
| | | * **복잡성:** 시스템이 복잡해져 전문 지식을 요구함. |
| | | * **오버헤드(Overhead):** DBMS의 기능(보안, 제어 등) 수행에 필요한 추가적인 처리 시간 발생. |

### 4. 데이터베이스 시스템의 구성 요소

| 슬라이드 | 제목 | 내용 구성 |
| :--- | :--- | :--- |
| **S 4.1** | **데이터베이스 시스템의 구성 요소** | * **구조 다이어그램 제시:** 데이터 $\leftrightarrow$ DBMS $\leftrightarrow$ 사용자/응용 프로그램 |
| | | 1. **데이터베이스 (DB):** 실제 저장된 데이터 집합. |
| | | 2. **DBMS:** DB를 관리하는 소프트웨어. |
| | | 3. **데이터 언어 (SQL):** DB와의 소통 수단. |
| | | 4. **사용자:** DBA, 일반 사용자, 응용 프로그래머 등. |
| **S 4.2** | **주요 DBMS 종류** | * **관계형 DBMS (RDBMS) 예시:** Oracle, MySQL, PostgreSQL, SQL Server |
| | | * **비관계형 DBMS (NoSQL) 소개:** MongoDB, Cassandra (잠깐 언급 후 심화 챕터에서 다룰 예정) |
