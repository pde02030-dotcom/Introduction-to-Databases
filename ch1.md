### 2. 데이터와 정보

| 슬라이드 | 제목 | 내용 구성 |
| :--- | :--- | :--- |
| **S 2.1** | **데이터(Data)의 정의** | * **정의:** 현실 세계에서 단순한 관찰이나 측정을 통해 수집된 **가공되지 않은 사실**이나 값. |

| | | * **예시:** "홍길동 학생의 키는 **175cm**이며, 오늘(2025-11-13) 서울의 **평균 기온은 10°C**다." |

| | | * **핵심:** 데이터베이스는 이 **데이터를 효율적으로 수집하고 보존**하는 역할을 한다. |

### 3. 데이터 관리의 변천사: 파일 시스템의 한계

| 슬라이드 | 제목 | 내용 구성 |
| :--- | :--- | :--- |
| **S 3.1** | **파일 시스템 (File System)** | * **정의:** 데이터를 응용 프로그램별로 개별 파일에 저장하고 관리하는 초기 방식. |
| | | * **구조:** 인사 관리 파일, 재고 관리 파일 등 각 프로그램이 독립적인 파일을 가짐. |
| **S 3.2** | **문제점 1: 데이터 중복 (Data Redundancy)** | * **설명:** 동일한 데이터(예: '학생 주소')가 여러 파일에 중복되어 저장됨. |
| | | * **결과:** 저장 공간 낭비, 데이터 관리 비용 증가. |
| **S 3.3** | **문제점 2: 데이터 비일관성 (Inconsistency)** | * **설명:** 중복된 데이터 중 일부만 수정되어 데이터가 서로 **모순**되는 현상 발생. |
| | | * **예시:** 인사 파일에는 '서울' 주소로, 급여 파일에는 '부산' 주소로 저장된 경우. |
| **S 3.4** | **문제점 3: 데이터 종속성 (Data Dependency)** | * **설명:** 응용 프로그램이 데이터 파일의 구조(레코드 형식, 저장 방식 등)에 **의존**하는 현상. |
| | | * **결과:** 데이터 파일 구조 변경 시, 해당 파일에 접근하는 모든 응용 프로그램 코드를 수정해야 함. (유지보수 어려움) |
| **S 3.5** | **문제점 4: 동시성 및 보안 취약** | * **설명:** 여러 사용자가 동시에 데이터를 접근하기 어렵고, 데이터에 대한 중앙 집중적인 **보안 및 권한 제어**가 불가능함. |

### 4. 데이터베이스 (DB)의 정의 및 특징

| 슬라이드 | 제목 | 내용 구성 아이디어 |
| :--- | :--- | :--- |
| **S 4.1** | **DBMS (Database Management System)의 필요성** | * **결론:** 파일 시스템의 문제점(중복, 비일관성, 종속성)을 극복하고 대규모 데이터를 효율적으로 관리하기 위해 **DBMS**가 등장. |
| **S 4.2** | **데이터베이스 (DB)의 정의** | * **4가지 핵심 요소 강조:** |
| | | 1. **통합된 데이터 (Integrated Data):** 최소의 중복으로 통합된 데이터. |
| | | 2. **저장된 데이터 (Stored Data):** 컴퓨터 저장 매체에 저장된 데이터. |
| | | 3. **운영 데이터 (Operational Data):** 조직의 고유한 업무 수행을 위해 반드시 필요한 데이터. |
| | | 4. **공용 데이터 (Shared Data):** 여러 사용자 및 응용 프로그램이 공동으로 소유하고 이용하는 데이터. |
| **S 4.3** | **데이터베이스의 주요 특징** | * **1. 실시간 접근성:** 질의에 대한 결과를 지연 없이 즉시 응답. |
| | | * **2. 계속적인 변화:** 새로운 데이터 삽입, 기존 데이터 삭제/수정 등 동적인 변화를 수용. |
| | | * **3. 동시 공유:** 여러 사용자가 동시에 동일한 데이터에 접근 가능. |
| | | * **4. 내용 기반 참조:** 데이터의 **물리적 위치**가 아닌 데이터의 **값(내용)**을 기준으로 검색 (예: '홍길동'을 찾아라). |
